## 1. 🏃서비스 소개

![Shot](https://github.com/FRONTENDSCHOOL7/final-06-jubging/assets/85389685/16502292-4f77-4f35-94a6-a60ae608dcdd)

줍깅은 **플로깅 경험을 공유하고, 플로깅 추천 코스를 공유**하는 SNS 플랫폼입니다.
'(쓰레기를) 줍다'와 '조깅(jogging)'을 합친 '줍깅'은 흔히 플로깅이라고도 불리우며, 환경 보호와 건강한 라이프스타일을 동시에 추구하는 컨셉을 가지고 있습니다.

- Follower, Following 기능을 활용해, 친구를 맺은 사용자들은 자신의 플로깅 경험을 기록하여 공유하고, 친구의 플로깅 경험도 느낄 수 있습니다.
- 자신의 홈 피드에서 팔로우한 사람의 게시글을 확인할 수 있고, 댓글과 좋아요 기능을 통해 서로 소통할 수 있습니다.
- 본인이 경험한 좋은 플로깅 코스를 공유할 수 있어 더 효율적인 플로깅 활동을 위한 데이터를 제공합니다. 본인의 플로깅을 공유하고 타인의 플로깅을 경험함으로써 환경 보호에 대한 인식을 높일 수 있습니다.

#### Recoil 활용 : 원자적 상태 관리와 코드 가독성 향상

```
· 원자적 상태관리를 통한 효율적인 협업 환경 구축
  - 각 상태가 독립적이며, 필요한 컴포넌트에만 상태 변경을 반영할 수 있도록하여 불필요한 랜더링을 줄일 수 있습니다
  - 각 팀원이 개별 상태를 독립적으로 관리할 수 있어 복잡한 상태 관리 코드를 줄이고 서로 작업에 덜 방해받는 환경을 만들 수 있었습니다

· 상태 가독성 및 유지보수성
  - Recoil은 상태 관리를 위한 복잡한 코드나 추가적인 라이브러리 없이도 상태관리를 명확하고 직관적으로 할 수 있게 도와준다 이로 인해 코드의 가독성과 유지보수성이 향상됩니다

· 직관적인 API, React와의 친화성
  - Recoil은 React의 특성을 최대한 활용하여 직관적이고 간결한 API를 제공하기 때문에 제공하기 때문에 상태 관리 로직의 복잡성을 줄여 코드 가독성을 향상 시킬 수 있습니다
  - React의 핵심 개념과 잘 어울리는 API를 제공하기 때문에 React를 배운지 얼마 안 된 팀원들도 Recoil에 어렵지 않게 적응할 수 있습니다
```

#### Axios 활용 : 비동기 요청 처리와 전역 설정 용이성

```
· Promise 기반 API 제공
  - Axios는 Promise 기반의 API를 제공하기 때문에 비동기 요청을 쉽게 처리할 수 있습니다
  - async/await 문법과 함께 사용하여 코드를 더욱 간결하고 읽기 쉽게 만들 수 있습니다

· 전역 설정
  - Axios는 전역적으로 설정할 수 있는 기능을 제공합니다 이를 통해 API의 기본 URL, 헤더, 타임아웃 시간 등을 한 곳에서 설정하고 관리할 수 있습니다

· 요청과 응답 Interceptor
  - Axios는 요청이나 응답을 가로채어 그 전후에 코드를 실행하도록 해주는 Interceptor 기능을 제공합니다 이를 통해 모든 요청이나 응답에 공통적으로 적용해야 하는 토큰 인증, 에러 처리 등과 같은 로직을 중앙에서 관리할 수 있습니다
```

#### CustomHook 활용 : 사용자 입력 및 스크롤 제어

```
ⅰ. useDebounce Hook
  - 사용자의 연속적인 입력을 제어하기 위해 생성한 Hook
  - 입력이 끝날 때까지 기다린 후, 해당 값을 반환하여 불필요한 API 호출을 줄이는 데 사용합니다

ⅱ. useScrollTop Hook
  - 스크롤 위치 초기화
  - 라우팅 간 페이지 전환 시 스크롤 위치를 페이지 상단으로 초기화하여, 사용자가 새로운 페이지의 내용을 제대로 볼 수 있게 합니다
```

#### React-Portal 활용 : 모달 컴포넌트 외부 렌더링 관리

```
react-portal을 사용하면, 모달 컴포넌트를 루트 외부에 렌더링 할 수 있어 CSS 스타일링과 이벤트 버블링에서 자유롭게 작업할 수 있습니다
```
